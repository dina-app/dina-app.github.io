Admin Toolkit for Salesforce — User Manual

How to install the toolkit, launch its apps from the popup, and use each workspace and tool.

Screenshots were taken against a Salesforce Developer Edition org; org-identifying values (host, org and user names, IDs, addresses) are replaced with sample values. This extension is not affiliated with, endorsed by, or sponsored by Salesforce.

1. Getting started & popup

  1. Install Admin Toolkit for Salesforce from the Chrome Web Store.
  2. Open any Salesforce tab and sign in as usual. The toolkit reuses the session you are already signed in with — no separate login.
  3. Click the extension icon to open the popup.

Apps tab

The Apps tab is the main entry point. It launches the four primary workspaces — Metadata Admin, Force Sheet, Org Review, and Object Schema — against the connected org.

Tools tab

The Tools tab lists the single-purpose pages: REST API, Data Export, Document Folders, Run Apex, Setup Audit Trail, Limits, Login History, Event Log Files, and the built-in Manual.

Org tab

The Org tab summarizes the connected org and the signed-in user.

Metadata search

The search box above the tabs finds metadata across the org by API name, label, or text, with results grouped and ranked.

2. Metadata Admin for Salesforce

Browse the org's metadata in a source-style tree, inspect folders and files, search the loaded workspace, track local edits, and build package.xml selections.

Workspace tree

Select workspace folders and namespaces in the header, then click Reload Workspace. The tree mirrors a source-format project: Apex classes, triggers, flows, Lightning Web Components, objects, and more.

Folders and files

Clicking a folder lists its records in a sortable table; clicking a row opens the file for inspection and local editing.

Search

The search box matches file and folder API names, labels, and content snippets within the loaded workspace.

Namespaces

The Workspace Namespaces dropdown scopes the workspace to unmanaged metadata or the managed namespaces you pick.

Changed files and package.xml

The Changed tab tracks your local edits so you can review, deploy, or discard them. The Package tab assembles a package.xml from tree selections.

3. Force Sheet

A multi-tab spreadsheet for Salesforce data: run SOQL, open reports and list views, import CSV/XLSX, and start CRUD workflows from grid selections — every write asks for confirmation first.

Query

Open the Query panel, type SOQL, and press Run (or Ctrl+Enter). Results stream into the grid with pagination.

Reports and uploads

The Report tab opens org reports (including joined reports) into the sheet. The Upload tab imports CSV or XLSX files.

CRUD workflows

The CRUD panel turns grid selections into insert, update, or delete requests. Every operation shows a confirmation with the exact records before anything is sent.

Workspace

The Workspace panel holds your files: auto-saved spreadsheet files in folders, the Objects tree (objects with list views), and the Reports tree. Files and folders export as CSV or Excel.

4. Org Review

A read-only assessment of org health that runs automatically when opened. Each section carries evidence and remediation guidance; the report exports as PDF, Markdown, summary JSON, or evidence JSON.

Collectors

Collector progress shows every read-only collector that contributed to the report.

Report sections

5. Object Schema

Build an interactive relationship map for any pickable object, then explore every field of every selectable object in the Fields spreadsheet below it.

Build controls

When the page opens, the Fields sheet already lists all fields of the pickable catalog. Packaged Objects and Show all standard objects control which objects are pickable; the packaged-namespace choice is remembered per org.

The relationship map

Press Build Schema View. Parents render on one side, children on the other; master-detail and lookup edges are distinguished. Click any node to see its relationships and focus the Fields sheet on it.

The Fields sheet

Filter to one object with Object filter, use native header filters, or run Global search — matching cells are highlighted. Download the current view as CSV or expand to full screen.

Layouts and alternative views

6. Document Folders

Browse Salesforce libraries and folders, preview supported files, and download selections as a ZIP that preserves folder paths. Folder selection is recursive; duplicate document occurrences stay independent. Downloads are limited to 500 files and 250 MB of known content and can be canceled mid-run.

7. REST Explorer

Compose Salesforce REST API requests with endpoint templates, send them against the connected org, and inspect the JSON response. Requests run only when you send them.

8. Data Export

Run SOQL, SOSL, and GraphQL queries or report exports, preview the result as a CSV grid or JSON, and download it.

9. Run Apex

Execute anonymous Apex and review the compile result, execution status, and debug output.

10. Limits & monitoring

Read-only monitoring pages for the org: API limits with live usage bars, the setup audit trail, login history, and event log files.

11. Built-in manual (EN/JP)

The extension ships with its own manual in English and Japanese, including per-version release notes. Every page has a language picker in its header, so the whole UI switches between English and 日本語 at any time.

12. Release notes

Recent releases. No release has added new Chrome permissions, host permissions, remote code, telemetry, ads, or third-party data services. The full per-version history ships inside the extension's built-in manual.

0.6.2 (2026-07-15)

  • Document Folders: folder selection is now recursive and atomic across lazily loaded subtrees, and duplicate document occurrences stay independent with folder-preserving ZIP paths.
  • Document Folders: added 500-file and 250 MB known-size ZIP limits, cancelable downloads, and one-time fetching of shared document content.
  • Object Schema: rebuilt the fields experience as a spreadsheet view with native header filters, global search highlights, CSV export, and full screen; Packaged Objects defaults to none and is remembered per org.
  • Object Schema became a popup Apps-only page with the flat compact header; the same 40px app-session header now also applies to Metadata Admin and Org Review.
  • The browser action and side panel stay available on every top-level toolkit page.

0.6.1 (2026-07-14)

  • Rebuilt Document Folders downloads around tree checkbox selection with a Selected Items panel; folder actions moved to the tree context menu.
  • Added Object Schema to the popup Apps tab: object suggestions, packaged-namespace and visible-object controls, synchronized map search with red relationship-path labels, zoom, horizontal/vertical layouts, and a field Value column.
  • Removed Org Review from the tools-page navigation (it opens from the popup Apps tab).
  • Refined English and Japanese interface terminology; removed all external status-service requests from Org Review.
  • Added THIRD-PARTY-NOTICES.txt with license attributions for the bundled spreadsheet engine.

0.6.0 (2026-07-11)

  • Added the read-only Org Review covering org health, limits, setup changes, login failures, and event-log availability.
  • Added a per-org read-only lock and consistent risk confirmation for Apex, data mutation, and metadata deployment.
  • Added metadata Validate Only and sanitized local operation receipts.
  • Fixed Force Sheet workspace files sharing the last active grid; values and formulas now stay isolated per file and org.
  • Improved Document Folders preview, large-file gating, named downloads, ZIP structure, and partial-failure handling.
