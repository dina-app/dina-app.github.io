Admin Toolkit for Salesforce ユーザーマニュアル

インストールから、ポップアップでのアプリ起動、各ワークスペース・ツールの使い方までを解説します。

スクリーンショットは Salesforce Developer Edition 組織で撮影し、組織を特定できる情報（ホスト名、組織名・ユーザー名、ID、アドレス）はサンプル値に置き換えています。本拡張機能は Salesforce の提携・承認・後援を受けていません。

1. はじめに・ポップアップ

  1. Chrome ウェブストアから Admin Toolkit for Salesforce をインストールします。
  2. いつもどおり Salesforce のタブを開いてサインインします。拡張機能はサインイン済みのセッションをそのまま利用するため、別途ログインは不要です。
  3. 拡張機能のアイコンをクリックしてポップアップを開きます。

アプリタブ

アプリタブがメインの入口です。Metadata Admin、Force Sheet、Org Review、Object Schema の 4 つの主要ワークスペースを、接続中の組織に対して起動します。

ツールタブ

ツールタブには単機能ページの一覧が並びます：REST API、データエクスポート、ドキュメントフォルダー、Apex 実行、設定変更履歴、制限値、ログイン履歴、イベントログファイル、内蔵マニュアル。

組織タブ

組織タブでは、接続中の組織とサインイン中のユーザーの概要を確認できます。

メタデータ検索

タブ上部の検索ボックスから、API 参照名・表示ラベル・テキストで組織全体のメタデータを検索できます。結果はグループ化・ランク付けされます。

2. Metadata Admin for Salesforce

組織のメタデータをソース形式のツリーで参照し、フォルダーやファイルの確認、ワークスペース内の検索、ローカル編集の追跡、package.xml の組み立てができます。

ワークスペースツリー

ヘッダーでワークスペースフォルダーと名前空間を選び、Reload Workspace をクリックします。ツリーはソース形式プロジェクトと同じ構造で、Apex クラス、トリガー、フロー、Lightning Web コンポーネント、オブジェクトなどが並びます。

フォルダーとファイル

フォルダーをクリックするとレコードが並べ替え可能な表で表示され、行をクリックするとファイルを開いて確認・ローカル編集できます。

検索

検索ボックスは、読み込み済みワークスペース内のファイル・フォルダーの API 参照名、表示ラベル、内容スニペットにマッチします。

名前空間

Workspace Namespaces ドロップダウンで、未管理メタデータのみ、または選択した管理パッケージ名前空間にワークスペースを絞り込めます。

変更ファイルと package.xml

Changed タブはローカル編集を追跡し、確認・デプロイ・破棄ができます。Package タブはツリーの選択から package.xml を組み立てます。

3. Force Sheet

Salesforce データのためのマルチタブ・スプレッドシートです。SOQL の実行、レポートやリストビューの展開、CSV/XLSX の取り込み、グリッド選択からの CRUD 操作ができます。書き込みは必ず実行前に確認ダイアログが表示されます。

クエリ

Query パネルを開いて SOQL を入力し、Run（または Ctrl+Enter）で実行します。結果はページネーション付きでグリッドに読み込まれます。

レポートとアップロード

Report タブは組織のレポート（結合レポートを含む）をシートに展開します。Upload タブは CSV / XLSX ファイルを取り込みます。

CRUD 操作

CRUD パネルはグリッドの選択範囲から挿入・更新・削除リクエストを作成します。送信前に、対象レコードを明示した確認が必ず表示されます。

ワークスペース

Workspace パネルにはあなたのファイルが保存されます。フォルダー分けされた自動保存スプレッドシートファイル、リストビューを持つオブジェクトの Objects ツリー、Reports ツリーがあり、ファイルやフォルダーは CSV / Excel でエクスポートできます。

4. Org Review（組織レビュー）

ページを開くと自動で実行される、読み取り専用の組織健全性アセスメントです。各セクションに根拠と改善ガイダンスが付き、レポートは PDF・Markdown・サマリー JSON・エビデンス JSON でエクスポートできます。

コレクター

Collector progress には、レポートに寄与したすべての読み取り専用コレクターが表示されます。

レポートセクション

5. Object Schema（オブジェクトスキーマ）

選択可能な任意のオブジェクトからインタラクティブなリレーションマップを作成し、その下の Fields スプレッドシートで、選択可能な全オブジェクトの全項目を探索できます。

作成コントロール

ページを開いた時点で、Fields シートには選択可能カタログの全項目が読み込まれます。Packaged Objects と Show all standard objects で選択可能なオブジェクトを制御でき、パッケージ名前空間の選択は組織ごとに記憶されます。

リレーションマップ

スキーマ図を作成を押すと、親オブジェクトと子オブジェクトが左右（または上下）に配置され、主従関係と参照関係が区別して描画されます。ノードをクリックするとリレーション詳細が表示され、Fields シートがそのオブジェクトに切り替わります。

Fields シート

Object filter で 1 オブジェクトに絞り込み、ネイティブのヘッダーフィルターを使い、Global search で全体を検索できます。一致セルはハイライトされます。表示中の内容は CSV でダウンロードでき、全画面表示にも切り替えられます。

レイアウトと代替ビュー

6. ドキュメントフォルダー

Salesforce のライブラリとフォルダーを参照し、対応ファイルをプレビューし、フォルダーパスを保持した ZIP として選択内容をダウンロードできます。フォルダー選択は再帰的で、同一ドキュメントの複数フォルダー配置も別々に扱われます。ダウンロードは 500 ファイル・既知サイズ 250 MB までで、実行中にキャンセルできます。

7. REST Explorer

エンドポイントテンプレートで Salesforce REST API リクエストを組み立て、接続中の組織に送信し、JSON レスポンスを確認できます。リクエストはあなたが送信したときだけ実行されます。

8. データエクスポート

SOQL・SOSL・GraphQL クエリやレポートエクスポートを実行し、結果を CSV グリッドまたは JSON でプレビューしてダウンロードできます。

9. Apex 実行

匿名 Apex を実行し、コンパイル結果・実行ステータス・デバッグ出力を確認できます。

10. 制限値とモニタリング

組織の読み取り専用モニタリングページ：使用量バー付きの API 制限、設定変更履歴、ログイン履歴、イベントログファイル。

11. 内蔵マニュアル（英語/日本語）

拡張機能には英語・日本語の内蔵マニュアルとバージョンごとのリリースノートが同梱されています。各ページのヘッダーには言語ピッカーがあり、UI 全体をいつでも英語と日本語で切り替えられます。

12. リリースノート

最近のリリースの概要です。いずれのリリースでも、新しい Chrome 権限・ホスト権限・リモートコード・テレメトリー・広告・サードパーティのデータサービスは追加されていません。バージョンごとの完全な履歴は拡張機能の内蔵マニュアルに同梱されています。

0.6.2（2026-07-15）

  • ドキュメントフォルダー：フォルダー選択が遅延読み込みサブツリーを含めて再帰的・アトミックになり、同一ドキュメントの複数フォルダー配置はフォルダーパスを保持した ZIP パスで別々に扱われます。
  • ドキュメントフォルダー：500 ファイル・既知サイズ 250 MB の ZIP 上限、ダウンロードのキャンセル、共有ドキュメント内容の一括取得（1 回のみ）を追加。
  • Object Schema：項目一覧をスプレッドシートビューとして再構築。ネイティブのヘッダーフィルター、全体検索のセルハイライト、CSV エクスポート、全画面表示に対応。Packaged Objects は既定で「なし」となり、組織ごとに記憶されます。
  • Object Schema はポップアップのアプリタブ専用ページとなり、フラットでコンパクトなヘッダーを採用。同じ 40px アプリセッションヘッダーを Metadata Admin と Org Review にも適用。
  • すべてのトップレベルページでブラウザーアクションとサイドパネルが利用可能になりました。

0.6.1（2026-07-14）

  • ドキュメントフォルダーのダウンロードをツリーのチェックボックス選択と選択済み項目パネル中心に再構築。フォルダー操作はコンテキストメニューに移動。
  • ポップアップのアプリタブに Object Schema を追加：オブジェクトサジェスト、パッケージ名前空間・表示オブジェクトの制御、赤いリレーションパスラベル付きのマップ検索、ズーム、横型/縦型レイアウト、項目の値列。
  • ツールページのナビゲーションから Org Review を削除（ポップアップのアプリタブから開きます）。
  • 英語・日本語の用語を改善（例：設定変更履歴）。Org Review から外部ステータスサービスへのリクエストを削除。
  • 同梱スプレッドシートエンジンのライセンス表記 THIRD-PARTY-NOTICES.txt を追加。

0.6.0（2026-07-11）

  • 組織の健全性・制限値・設定変更・ログイン失敗・イベントログを対象とする読み取り専用の Org Review を追加。
  • 組織ごとの読み取り専用ロックと、Apex・データ変更・メタデータデプロイに対する一貫したリスク確認を追加。
  • メタデータの検証のみ（Validate Only）デプロイとサニタイズ済みのローカル操作履歴を追加。
  • Force Sheet のワークスペースファイルが最後のグリッドを共有してしまう問題を修正。値と数式はファイル・組織ごとに分離されます。
  • ドキュメントフォルダーのプレビュー、大容量ファイルの制御、ダウンロード名、ZIP 構造、部分失敗の処理を改善。
