DinaLab Support

Use the public issue tracker for product questions, bug reports, privacy questions, and documentation feedback.

Open a support issue

Before submitting

  • Include the product name, version, Chrome version, and steps to reproduce the problem.
  • Remove Salesforce session IDs, cookies, authorization headers, access tokens, personal data, and confidential org data.
  • For a security-sensitive report, do not post secrets or exploit details in a public issue.

DinaLab does not currently publish a separate security-reporting email. A verified publisher contact email must still be configured in the Chrome Web Store dashboard before release.
